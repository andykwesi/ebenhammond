# Literature Review: Credit Card Fraud Detection Using Machine Learning

## Introduction

Credit card fraud detection has become an important research and engineering problem because electronic payments now mediate a large share of everyday economic activity. As card-not-present transactions, online retail, mobile banking, and digital wallets have expanded, financial institutions have had to identify fraudulent activity among very large volumes of legitimate transactions. The problem is not simply to classify transactions as fraudulent or genuine; it is to do so quickly, accurately, and in a way that balances customer experience, operational investigation cost, and financial loss. A fraud detection system that blocks too many legitimate transactions may frustrate customers and impose avoidable review costs, while a system that misses fraudulent transactions exposes cardholders and institutions to loss. This makes fraud detection a high-stakes binary classification problem with strongly asymmetric costs (Bhattacharyya et al., 2011; Dal Pozzolo et al., 2018).

Machine learning is widely used for fraud detection because fraudulent behavior is adaptive and difficult to capture completely with static rules. Traditional rule-based systems can encode known fraud patterns, such as unusually large purchases, geographically inconsistent transactions, or repeated failed attempts. However, rule sets tend to become brittle as fraudsters change their strategies, and they may not capture subtle relationships among transaction attributes. Supervised learning models can learn patterns from labeled historical transactions and generate fraud probabilities or risk scores for new transactions. These scores can then support automated decisions, alerts, or manual investigation workflows. In the credit card setting, supervised learning is especially attractive because card issuers and payment processors often accumulate large histories of labeled transactions, although those labels may arrive with delay after chargebacks or investigation outcomes (Dal Pozzolo et al., 2018; Lucas & Jurgovsky, 2020).

Despite this suitability, credit card fraud detection remains difficult for several reasons. First, the target class is extremely rare. In the widely used European credit card fraud dataset, only 492 of 284,807 transactions are fraudulent, meaning fraud accounts for approximately 0.172% of all observations (Machine Learning Group - ULB, n.d.; OpenML, n.d.). A classifier that predicts every transaction as legitimate would achieve very high accuracy while detecting no fraud at all. Second, financial transaction data are often anonymized or privacy-protected, which limits interpretability and feature engineering. The European dataset, for example, includes 28 principal components labeled `V1` through `V28`, plus `Time`, `Amount`, and the target variable `Class`, but it does not reveal the original transaction features (Machine Learning Group - ULB, n.d.). Third, fraud is nonstationary. Customer behavior changes over time, seasonal patterns alter purchase distributions, and fraudsters adapt in response to security controls. This phenomenon is a form of concept drift, where the relationship between features and labels changes after a model is trained (Gama et al., 2014).

The present project addresses this problem through a thesis-ready Flask prototype that trains and compares several classical machine learning models for credit card fraud detection. The implemented system uses Logistic Regression, Random Forest, Support Vector Machine, and Gradient Boosting classifiers; handles class imbalance with SMOTE where enough minority-class examples are available; and evaluates models with fraud-sensitive metrics including precision, recall, F1-score, ROC-AUC, PR-AUC, and confusion matrix counts. This literature review therefore focuses on research that directly informs the design of such a system: the fraud detection context, common datasets, machine learning approaches, imbalanced-data strategies, evaluation metrics, comparative algorithm performance, and remaining research gaps. The review argues that classical supervised models remain valuable for credit card fraud detection prototypes because they are interpretable enough for academic comparison, computationally practical, and compatible with the severe imbalance and metric requirements of the domain.

## Credit Card Fraud Detection Context

Credit card fraud detection is usually framed as a transaction-level classification task. Each transaction is represented by a vector of features, and the model predicts whether the transaction belongs to the legitimate class or the fraud class. This formulation is useful because it maps naturally to standard supervised learning algorithms and to operational workflows where each incoming transaction must be accepted, declined, scored, or sent for review. However, the apparent simplicity of the binary classification setup hides a number of practical complications. Fraud is rare, labels may be delayed, feature distributions change, and the business cost of false negatives and false positives is not equal (Dal Pozzolo et al., 2018; Lucas & Jurgovsky, 2020).

The rarity of fraud is one of the defining features of the domain. He and Garcia (2009) describe imbalanced learning as a situation in which one class is heavily underrepresented relative to another, making ordinary classification algorithms biased toward the majority class. In credit card fraud detection, legitimate transactions greatly outnumber fraudulent ones. This imbalance creates two linked problems. The first is statistical: models may see too few minority examples to learn robust fraud patterns, especially when fraud behavior itself is diverse. The second is evaluative: aggregate metrics such as accuracy can give a misleading impression of performance. If fraud accounts for less than one percent of transactions, a model can achieve more than 99% accuracy while failing at the central task of detecting fraud.

Credit card fraud also differs from many benchmark classification tasks because it is embedded in an adversarial environment. Fraudsters actively attempt to avoid detection, and their strategies evolve as institutions introduce new controls. Dal Pozzolo et al. (2018) identify class imbalance, concept drift, and verification latency as central challenges in realistic fraud detection systems. Verification latency refers to the delay between a transaction and the availability of a reliable label, since a transaction may only be confirmed as fraudulent after customer complaint, chargeback, or investigation. This means that models are often trained on labels that lag behind current fraud behavior. Gama et al. (2014) describe concept drift more generally as a change in the relationship between input features and target labels over time. In fraud detection, drift can arise from changes in consumer habits, seasonal spending, new merchant categories, new attack methods, and the fraudsters' response to deployed models.

Researchers have also emphasized the importance of feature representation. Whitrow et al. (2009) show that transaction aggregation can improve credit card fraud detection by summarizing behavior over a window of previous transactions, rather than treating each transaction as an isolated event. Aggregated features can capture velocity, frequency, spending deviation, and other behavior patterns that are difficult to infer from a single transaction. Lucas and Jurgovsky (2020) similarly note that credit card fraud detection can benefit from temporal and sequential information, including transaction histories and engineered aggregates. The public European dataset used in many academic projects does not expose raw behavioral variables, but its `Time` and `Amount` fields, combined with anonymized principal components, still support the study of severe imbalance and model comparison.

The European credit card fraud dataset has become a common benchmark because it is public, real-world, and highly imbalanced. It contains transactions made by European cardholders over two days in September 2013, with 284,807 transactions and 492 fraud cases (Machine Learning Group - ULB, n.d.; OpenML, n.d.). Most input variables are the result of principal component analysis, which protects privacy but limits semantic interpretation. The target variable `Class` uses `1` for fraud and `0` for legitimate transactions. This schema matches the present project's expected input: `Time`, `V1` through `V28`, `Amount`, and `Class`. The dataset is therefore well suited to a thesis prototype because it permits reproducible experimentation without exposing sensitive payment data. Its limitations are equally important: it spans only two days, lacks raw feature meanings, and may not represent the full variety of modern payment fraud.

Because real-world fraud detection is costly and dynamic, the goal of a model is not merely to maximize an abstract score. A useful system must identify enough fraudulent transactions to reduce loss while limiting the number of legitimate transactions incorrectly flagged. False negatives allow fraud to proceed, whereas false positives can inconvenience customers and burden investigators. The relative cost of these errors depends on institutional policy, transaction value, available review capacity, and regulatory requirements. This cost asymmetry explains why fraud detection literature often discusses recall, precision, PR-AUC, thresholds, and cost-sensitive evaluation rather than relying only on accuracy or ROC-AUC (Dal Pozzolo et al., 2018; Saito & Rehmsmeier, 2015).

## Machine Learning for Fraud Detection

Machine learning methods for credit card fraud detection can be grouped broadly into supervised, unsupervised, semi-supervised, and hybrid approaches. Supervised methods learn from labeled examples of legitimate and fraudulent transactions. Unsupervised methods seek anomalies without requiring fraud labels, which can be useful when labels are sparse or delayed. Semi-supervised methods often learn the structure of legitimate behavior and flag deviations. Hybrid approaches combine rules, anomaly scores, supervised classifiers, and human review. The current project uses supervised learning because it assumes access to labeled transaction data and aims to compare established classifiers in a transparent academic prototype.

Supervised learning is a natural starting point when labeled transaction data are available. Bhattacharyya et al. (2011) compared Logistic Regression, Random Forest, and Support Vector Machines for credit card fraud detection and showed that multiple data mining techniques can be applied to the same fraud detection problem with different tradeoffs. Their work is relevant to this project because it supports the comparative design: no single algorithm should be assumed best without empirical evaluation on the target dataset and selected metrics. Khatri et al. (2020) similarly compared supervised machine learning algorithms for credit card fraud detection, reflecting a common research pattern in which multiple classifiers are trained and compared under shared preprocessing and evaluation conditions.

Logistic Regression is one of the simplest and most interpretable supervised classifiers. It models the log-odds of the positive class as a linear function of the input features and can output calibrated probabilities under appropriate assumptions and preprocessing. In fraud detection, Logistic Regression is useful as a baseline because it is fast, relatively transparent, and often competitive when the decision boundary is approximately linear after feature transformation. It also supports class weighting, allowing the minority fraud class to receive greater importance during training. However, its linear form may limit performance when fraud patterns depend on nonlinear interactions among variables. In the present project, Logistic Regression is combined with feature scaling and balanced class weights, which is consistent with its sensitivity to feature magnitude and class distribution.

Random Forest is an ensemble method that builds many decision trees and aggregates their predictions (Breiman, 2001). It can model nonlinear relationships and interactions without requiring the same scaling assumptions as linear models or kernel methods. Random Forest is often effective in tabular fraud detection because decision trees can partition feature space around complex combinations of transaction attributes. The ensemble structure reduces the variance of individual trees and can improve generalization. In the present system, Random Forest is configured with balanced class weights and a minimum leaf size, which helps reduce overfitting while addressing class imbalance. Its main limitations are reduced interpretability compared with Logistic Regression and the possibility that probability estimates may require calibration for decision-making.

Support Vector Machines are margin-based classifiers that seek a separating hyperplane with maximum margin, and kernel functions allow them to model nonlinear boundaries (Cortes & Vapnik, 1995). In fraud detection, SVMs can be useful when the boundary between legitimate and fraudulent transactions is complex but the dataset is not too large for kernel computation. The radial basis function kernel, used in the present project, is a common choice for nonlinear classification. SVMs are sensitive to feature scaling, so the system correctly includes standardization before SVM training. Their disadvantages include computational cost on large datasets, sensitivity to hyperparameters, and less direct interpretability. Probability outputs are typically derived through calibration procedures rather than being native to the basic margin formulation.

Gradient Boosting builds an additive ensemble of weak learners, often decision trees, where each new learner attempts to correct errors made by the previous ensemble (Friedman, 2001). Boosting methods are powerful for structured tabular data because they capture nonlinear relationships and interactions while focusing on difficult cases. In fraud detection, boosting can be effective because fraud examples may occupy complex regions of feature space. However, boosting can overfit if model complexity, learning rate, and tree depth are not controlled. The current project includes Gradient Boosting as a classical ensemble comparator. Unlike the other three implemented models, the current pipeline does not attach SMOTE to Gradient Boosting, which may be reasonable as a conservative baseline but also identifies a possible future experiment: comparing boosted models with and without resampling or class-weighted alternatives.

Recent literature also explores deep learning, graph learning, sequence models, and federated learning for fraud detection. Lucas and Jurgovsky (2020) review sequential methods such as recurrent neural networks and emphasize the importance of temporal dynamics. Deep learning can capture complex representations from large transaction histories, but it may require larger datasets, careful tuning, and stronger computational resources than classical methods. For a thesis prototype using the European dataset, classical supervised models provide a defensible scope. They allow clear comparison, simpler explanation, and direct alignment with core fraud detection issues: imbalance, metrics, preprocessing, and decision thresholds.

## Handling Class Imbalance

Class imbalance is central to credit card fraud detection. In the European dataset, the fraud class represents approximately 0.172% of transactions, meaning that a model trained naively may prioritize the majority class and perform poorly on the minority class (Machine Learning Group - ULB, n.d.). He and Garcia (2009) argue that imbalanced learning requires attention to data distribution, algorithm behavior, and evaluation metrics. This is especially true when the minority class is the class of greatest practical interest. In credit card fraud detection, the minority class is not noise; it is the target of the system.

One common response to imbalance is resampling. Undersampling reduces the number of majority-class examples, while oversampling increases the representation of minority-class examples. Simple random oversampling duplicates minority examples, but duplication can increase overfitting because the model repeatedly sees identical fraud cases. SMOTE, introduced by Chawla et al. (2002), addresses this by generating synthetic minority examples between existing minority observations. Rather than merely copying fraud cases, SMOTE creates new points in feature space by interpolating between minority-class neighbors. This can help a classifier learn a broader minority decision region.

SMOTE is widely used in fraud detection research because it is easy to apply and often improves minority-class recall. Ileberi et al. (2021) used SMOTE in a credit card fraud detection framework and evaluated several machine learning methods on an imbalanced European cardholder dataset. Their study reflects the broader research pattern of pairing resampling with model comparison. The present project follows this pattern by using SMOTE when enough fraud examples are available in the training split. The implementation choice to condition SMOTE on the number of fraud cases is important because synthetic neighbor-based oversampling can fail or behave poorly when the minority class has too few examples for the selected neighbor parameter.

Another common approach is class weighting. Instead of changing the dataset distribution, class weighting changes the learning objective so that errors on the minority class receive greater penalty. Logistic Regression, Random Forest, and SVM implementations often support balanced class weights. The current project uses `class_weight="balanced"` for these models, which complements SMOTE by encouraging the algorithm to treat fraud errors as more important than the raw class frequency would imply. Class weighting is attractive because it avoids generating synthetic data, but it does not create additional minority patterns. Resampling and class weighting therefore address imbalance from different angles: one changes the training data distribution, and the other changes the cost structure during learning.

Undersampling has also been studied in fraud detection, but it introduces its own risk. Dal Pozzolo et al. (2015) examine probability calibration with undersampling and show that changing class priors in the training set can bias posterior probability estimates. This issue matters because fraud detection systems often use probability scores or thresholds to decide which transactions require action. If resampling distorts probability estimates, a model may rank transactions usefully but produce poorly calibrated probabilities. This is one reason why fraud detection should distinguish ranking performance from decision calibration. A model may have good ROC-AUC or PR-AUC while still requiring threshold adjustment or calibration before deployment.

Class imbalance also interacts with validation strategy. Random train-test splits can overstate performance if the operational setting is temporal. Fraud detection systems are used prospectively: models trained on past transactions are applied to future transactions. If the data distribution changes over time, random splits may mix future-like patterns into training and produce optimistic estimates. Dal Pozzolo et al. (2018) emphasize realistic modeling conditions, including concept drift and verification latency. For a thesis prototype, stratified random splitting is acceptable for controlled comparison, but the literature suggests that future work should evaluate chronological splits, rolling windows, or drift-aware retraining when moving toward operational deployment.

The current project's imbalance handling is therefore defensible but should be presented as a prototype strategy rather than a complete production solution. SMOTE and class weights are appropriate for demonstrating awareness of severe imbalance and for improving sensitivity to fraud. However, they do not solve every imbalance-related issue. They do not automatically calibrate probabilities, define business thresholds, address temporal drift, or guarantee that synthetic examples reflect realistic fraud behavior. A strong literature review must recognize this distinction: imbalance handling improves the learning process, but fraud detection still requires careful evaluation and operational thresholding.

## Evaluation Metrics for Fraud Detection

Evaluation is one of the most important methodological choices in credit card fraud detection. Because the positive class is rare, accuracy is usually inappropriate as the main metric. In the European dataset, a model that predicts all transactions as legitimate would be approximately 99.8% accurate while having zero fraud recall. This contradiction illustrates why fraud detection requires metrics that separately examine the detection of fraud and the cost of false alerts. The present project appropriately reports precision, recall, F1-score, ROC-AUC, PR-AUC, and confusion matrix counts.

The confusion matrix provides the foundation for these metrics. True positives are fraudulent transactions correctly identified as fraud. False negatives are fraudulent transactions incorrectly classified as legitimate. True negatives are legitimate transactions correctly classified as legitimate. False positives are legitimate transactions incorrectly flagged as fraud. In financial terms, false negatives may correspond to direct fraud losses, while false positives may correspond to customer friction and investigation cost. The relative importance of these errors depends on the business context, but both are important. A model with high recall but extremely low precision may overwhelm investigators with false alarms, while a model with high precision but low recall may miss too many fraud cases.

Recall, also called sensitivity, measures the proportion of actual fraud cases that the model identifies. It is especially important when missing fraud is costly. The current project ranks models first by recall, which is a reasonable thesis design because the purpose of the system is to detect fraudulent transactions. However, recall alone is insufficient. A model can achieve high recall by flagging many legitimate transactions as suspicious. Precision measures the proportion of predicted fraud cases that are truly fraudulent, helping assess the quality of alerts. In operational fraud detection, precision is linked to review efficiency. A low-precision model may produce many false positives, increasing manual workload and customer disruption.

The F1-score combines precision and recall using their harmonic mean. It is useful when both missed fraud and false alerts matter, and it penalizes models that perform poorly on either dimension. The current project uses F1-score after recall in its model ranking, which reflects a practical compromise: prioritize catching fraud, then prefer models whose fraud alerts are more reliable. However, F1-score assumes equal weight between precision and recall. In real financial settings, the relative cost of false negatives and false positives may not be equal, so F1-score should not be treated as a universal business objective. Cost-sensitive evaluation or threshold optimization may be more appropriate when monetary loss estimates are available.

ROC-AUC measures the ability of a model to rank positive examples above negative examples across thresholds. It is widely used in binary classification and remains informative for comparing ranking performance. Davis and Goadrich (2006) explain the relationship between ROC and precision-recall curves and show that the two spaces are connected, but they also highlight important differences under skewed class distributions. In highly imbalanced datasets, ROC-AUC can appear strong even when precision is weak because the large number of true negatives can make false positive rates look small. For this reason, ROC-AUC should be interpreted alongside PR-AUC in credit card fraud detection.

PR-AUC, or area under the precision-recall curve, focuses directly on the positive class. Saito and Rehmsmeier (2015) argue that precision-recall plots are more informative than ROC plots when evaluating binary classifiers on imbalanced datasets. This is particularly relevant to fraud detection because the positive class is rare and operational attention is centered on fraud alerts. PR-AUC summarizes the tradeoff between precision and recall across thresholds. A high PR-AUC indicates that the model can maintain useful precision while detecting a meaningful share of fraud cases. The current project's inclusion of PR-AUC is therefore methodologically strong and aligned with imbalanced-learning literature.

Threshold selection is another important evaluation issue. Many classifiers output probabilities or scores, but the final classification depends on a threshold, often defaulting to 0.5. In fraud detection, a 0.5 threshold may be inappropriate because fraud is rare and the cost of missing fraud may be higher than the cost of review. Dal Pozzolo et al. (2015) show that sampling strategies can affect posterior probability estimates, and Dal Pozzolo et al. (2018) emphasize realistic performance measures for fraud detection. A prototype may compare default-threshold classifications for simplicity, but a production-oriented system should consider threshold tuning based on recall targets, review capacity, expected cost, or precision requirements.

The project's metric design reflects a good understanding of these issues. It reports multiple metrics rather than relying on a single score, and it ranks models using recall, F1-score, PR-AUC, and false positives. This ranking strategy explicitly recognizes that fraud detection is not a generic accuracy maximization problem. Nevertheless, the literature suggests two future improvements. First, the system could evaluate threshold curves rather than only fixed-threshold metrics. Second, it could add cost-sensitive evaluation if assumptions about average fraud loss and review cost are available. These additions would move the prototype closer to operational decision-making while retaining academic clarity.

## Comparative Discussion of Algorithms

The four implemented models represent a useful cross-section of classical supervised learning approaches. Logistic Regression provides a linear probabilistic baseline. Random Forest provides a bagged tree ensemble with nonlinear decision boundaries. Support Vector Machine provides a margin-based nonlinear classifier through the RBF kernel. Gradient Boosting provides a sequential ensemble that focuses on correcting previous errors. Comparing these models allows the project to demonstrate how algorithmic assumptions affect fraud detection performance under the same dataset and metric regime.

Logistic Regression is valuable because it establishes a transparent baseline. If a simple linear model performs competitively, this suggests that the transformed features contain linear separability useful for fraud detection. The European dataset's `V1` through `V28` features are principal components, which may already encode combinations of original variables. This may help a linear classifier perform better than it would on raw, heterogeneous transaction data. Logistic Regression also produces coefficients that can be inspected, although the anonymized PCA features limit substantive interpretation. Its weakness is that it cannot easily capture complex interactions unless those interactions are engineered into the features.

Random Forest is often attractive in fraud detection because it handles nonlinearities and interactions automatically. Bhattacharyya et al. (2011) included Random Forest in their comparative study, and fraud detection surveys frequently identify tree ensembles as strong performers on tabular financial data (Lucas & Jurgovsky, 2020). Random Forest can be robust to outliers and does not require feature scaling, making it convenient for mixed tabular data. In the present project, Random Forest benefits from class weighting and SMOTE, which may improve fraud sensitivity. However, Random Forest predictions can become less reliable when the future fraud distribution differs from the training data. The model also provides limited causal explanation: feature importance scores can be useful, but they are not equivalent to transparent decision rules.

Support Vector Machines offer a different tradeoff. With an RBF kernel, SVMs can construct flexible boundaries around minority-class regions. This can be useful when fraud cases cluster in unusual parts of feature space. Bhattacharyya et al. (2011) evaluated SVMs alongside Logistic Regression and Random Forest, reflecting their established role in fraud data mining. However, SVMs can be computationally expensive on large datasets and sensitive to hyperparameters such as `C` and kernel scale. Probability estimates require additional calibration. For a thesis prototype, SVM is still a valuable comparator because it introduces a margin-based approach distinct from linear and tree-based models.

Gradient Boosting is included because boosted tree models often perform well on structured classification tasks. Friedman (2001) describes gradient boosting as a method for building additive models through stagewise optimization. In fraud detection, boosting can identify difficult cases and refine decision boundaries over successive learners. Ileberi et al. (2021) studied SMOTE and AdaBoost in credit card fraud detection, showing that boosting-based methods are commonly explored in the field. The current project's Gradient Boosting classifier provides a conventional boosting baseline, though future work could compare it with more recent implementations such as XGBoost, LightGBM, or CatBoost. Such models often include regularization, missing-value handling, and optimized tree growth strategies that can improve performance on large tabular datasets.

A comparative design is important because the best fraud detection model depends on the dataset, preprocessing, imbalance strategy, and evaluation metric. A model optimized for accuracy may not be best for recall or PR-AUC. A model with high recall may produce too many false positives. A model with strong ROC-AUC may still have disappointing precision under severe imbalance. This is why the present project's model selection rule is appropriate: it does not simply select the most accurate model, but ranks by recall, then F1-score, then PR-AUC, then fewer false positives. That ordering reflects the fraud-sensitive priorities described in the literature.

There are also methodological considerations in comparing models. All models should be evaluated on the same held-out test split, and resampling should be applied only to the training data to avoid leakage. The current project uses an imbalanced-learn pipeline, which is a sound implementation pattern because it ensures that SMOTE is part of the training pipeline rather than being applied to the entire dataset before splitting. Leakage is a common risk in imbalanced classification: if synthetic examples are created before the train-test split, information from test observations can influence training data. By building SMOTE into the pipeline and fitting only on the training fold, the project follows better practice.

Scaling is another algorithm-specific issue. Logistic Regression and SVM require standardization because their objectives are sensitive to feature magnitude. Random Forest and Gradient Boosting are tree-based methods and generally do not require scaling. The current project uses `StandardScaler` for Logistic Regression and SVM but not for tree-based models, aligning preprocessing with algorithm requirements. This avoids unnecessary transformations for trees while supporting stable optimization and distance-based kernel behavior for models that need it.

Overall, the implemented algorithm set is suitable for a thesis chapter because it supports both practical experimentation and critical analysis. The models are familiar, well-supported, and sufficiently different from one another to generate meaningful comparison. The literature does not imply that these four models exhaust the field; rather, it supports them as a defensible classical baseline. More advanced research could incorporate cost-sensitive learning, anomaly detection, graph-based features, neural sequence models, or drift-aware online learning. However, those additions would also expand the complexity of the project. For the current system, the selected models strike a reasonable balance between rigor, explainability, and implementation feasibility.

## Research Gaps and Project Justification

The literature reveals several gaps that justify the present project while also defining its limitations. The first gap concerns the distance between benchmark experiments and operational fraud detection. Public datasets such as the European credit card fraud dataset are valuable for reproducibility, but they simplify real-world conditions. They often lack raw feature semantics, longer time spans, investigator feedback loops, and institution-specific cost structures. Dal Pozzolo et al. (2018) argue that realistic fraud detection must address class imbalance, concept drift, and verification latency together. A thesis prototype cannot fully reproduce a bank's live fraud detection environment, but it can demonstrate core modeling principles on a standard dataset and identify what would be required for deployment.

The second gap concerns evaluation practice. Many introductory fraud detection projects report accuracy even though accuracy is poorly suited to severe imbalance. Saito and Rehmsmeier (2015) and Davis and Goadrich (2006) support the use of precision-recall analysis for imbalanced binary classification. The current project addresses this gap by reporting PR-AUC, recall, precision, F1-score, and confusion matrix counts. Its ranking strategy further emphasizes recall and false positives rather than treating all errors equally. This makes the system more academically defensible than a prototype that reports only accuracy or ROC-AUC.

The third gap concerns imbalance handling. SMOTE is widely used, but it is sometimes applied mechanically without attention to data leakage, probability calibration, or the realism of synthetic examples. Chawla et al. (2002) introduced SMOTE as a way to create synthetic minority examples, and later fraud detection studies have used it to improve minority-class learning (Ileberi et al., 2021). However, Dal Pozzolo et al. (2015) show that sampling strategies can distort posterior probabilities. The current project uses SMOTE within a pipeline and combines it with class weighting for selected models, which is a reasonable educational and experimental design. The review should nevertheless make clear that future work should examine calibration and threshold optimization.

The fourth gap concerns temporal behavior. Fraud detection is not static. Gama et al. (2014) describe concept drift as a central issue in online supervised learning, and Dal Pozzolo et al. (2018) specifically apply this concern to credit card fraud detection. The current project uses a static train-test split, which is appropriate for a first prototype but not sufficient for real-time deployment. A future extension could compare random stratified validation with chronological validation, retrain models over rolling windows, and monitor changes in recall and PR-AUC over time. Such work would connect the prototype to realistic fraud operations.

The fifth gap concerns feature engineering and interpretability. Whitrow et al. (2009) show that transaction aggregation can improve fraud detection, and Lucas and Jurgovsky (2020) emphasize sequence and temporal modeling. The European dataset's PCA-transformed features limit feature engineering and explanation because the original variables are hidden. This privacy-preserving structure is useful for public research but constrains interpretability. The current project therefore focuses on algorithmic comparison and metric design rather than explaining specific transaction behaviors. If raw transaction histories were available, future work could add aggregated spending features, merchant-category patterns, device signals, geographic indicators, and customer-level baselines.

These gaps justify the project as a practical academic prototype. It does not claim to solve all challenges of production fraud detection. Instead, it demonstrates an end-to-end workflow: validate a standard transaction schema, train multiple classical models, handle imbalance with SMOTE and class weights, select a model using fraud-sensitive metrics, save artifacts, and expose predictions through a Flask dashboard. This workflow is valuable because many fraud detection studies remain either theoretical or notebook-based. A working application shows how literature-informed modeling choices can be translated into a usable system for single and batch prediction.

The project is also appropriate for a thesis because it supports critical comparison. Logistic Regression, Random Forest, SVM, and Gradient Boosting can be discussed in terms of assumptions, strengths, weaknesses, preprocessing requirements, and metric outcomes. SMOTE can be evaluated as an imbalance strategy rather than merely mentioned. PR-AUC and recall can be justified with reference to imbalanced evaluation literature. The dashboard and artifact storage make the system demonstrable, while the synthetic demo dataset fallback ensures that the application remains runnable even when the public dataset is absent. For final thesis results, however, the public European dataset should be used because the synthetic dataset is only a demonstration aid and does not represent real fraud behavior.

## Summary

The literature on credit card fraud detection shows that the problem is best understood as a rare-event, cost-sensitive, and temporally dynamic classification task. Machine learning is appropriate because fraud patterns are complex and adaptive, but model development must address class imbalance, evaluation bias, and operational constraints. Public datasets such as the European credit card fraud dataset enable reproducible experimentation, although their anonymized and time-limited structure restricts interpretability and deployment realism.

Classical supervised models remain useful for thesis-level fraud detection systems. Logistic Regression provides a transparent baseline, Random Forest captures nonlinear tabular patterns, SVM offers margin-based nonlinear classification, and Gradient Boosting supplies a strong sequential ensemble approach. The literature supports comparing such models rather than assuming a single best algorithm. Because fraud is rare, class imbalance strategies such as SMOTE and class weighting are necessary, but they must be applied carefully to avoid leakage and probability distortion.

Evaluation must focus on fraud-sensitive metrics. Accuracy is inadequate under extreme imbalance. Recall, precision, F1-score, ROC-AUC, PR-AUC, and confusion matrix counts provide a more complete view of performance. Precision-recall analysis is particularly important because it focuses on the rare positive class and better reflects the alert-quality problem faced by fraud detection systems. The current project aligns well with these findings by ranking models according to recall, F1-score, PR-AUC, and false positives.

The reviewed literature also identifies future directions. A more advanced system could incorporate chronological validation, threshold optimization, cost-sensitive learning, probability calibration, transaction aggregation, concept-drift monitoring, and deeper temporal models. These extensions would improve realism, but they are not required for a defensible thesis prototype. The existing system provides a clear, literature-informed foundation for credit card fraud detection using machine learning and demonstrates how academic concepts can be implemented in a practical Flask application.
